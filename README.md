# CS360-Portfolio

This project was focused on building a simple weight‑tracking Android app that lets users log their daily weight, update or delete entries, and set a personal goal. The main goal of the app was to give users a quick and easy way to track their progress without needing an online account or complicated features. The user need I focused on most was simplicity, people should be able to open the app, enter their weight, and see their progress without extra steps or clutter.

To support those needs, the app included a login screen, a main screen with a list of weight entries, an “add weight” screen, an “update weight” screen, and a goal‑setting screen. Each screen was designed to be straightforward, with clear buttons and minimal text so users wouldn’t get confused. The UI designs kept users in mind by keeping everything consistent, grouping related actions together, and making sure the most common tasks were easy to reach. This approach worked well because it kept the app focused on what users actually needed instead of adding unnecessary features.

When coding the app, I broke the work into small steps instead of trying to build everything at once. I started with the database, then wired up the UI screens, and finally added the SMS permission and goal‑reached feature. Working in small pieces made it easier to test and fix issues as I went. This strategy is something I’ll keep using in future projects because it keeps the development process organized and less overwhelming.

To make sure the app worked correctly, I tested each feature as soon as I built it, logging in, adding weights, updating entries, deleting entries, and checking that the goal notification worked. Testing early and often helped catch small mistakes before they turned into bigger problems. It also showed how important it is to test on an emulator that matches the Android version you’re targeting.

Throughout the whole process, the biggest challenge was getting all the parts, UI, database, and permissions, to work together smoothly. I had to adjust my design a few times and rethink how certain screens connected. That forced me to be flexible and find solutions that still kept the app simple. One area where I felt especially successful was the database and CRUD functionality. I was able to build a clean, reliable system for storing and updating user data, which is a core skill for mobile development.
